# LOT 불량 원인 분석 레포트 생성 양식 및 예시

## 1. 텍스트 레포트 양식 (OpenAI 프롬프트)

### 1-1. 시스템 프롬프트 (한글)

```
당신은 배터리 전극 제조 공정 분석 전문가입니다. 비전공자도 이해할 수 있도록 불량 LOT 분석 레포트를 작성해주세요. 레포트 제목이나 LOT ID를 본문에 다시 언급하지 마세요. 권장 조치사항은 반드시 번호(1. 2. 3.)를 사용하여 나열해주세요.
```

### 1-2. 사용자 프롬프트 구조 (한글)

```
다음 불량 LOT에 대한 불량 원인 분석 레포트를 작성해주세요. 반드시 아래 양식을 따르세요:

**불량 원인 분석**: "{가장 영향 큰 파라미터} (X.X%), {두 번째} (X.X%), {세 번째} (X.X%)" 형식으로 기여도를 나열하세요.

**불량 발생 메커니즘**:
각 파라미터별로 비전공자도 이해할 수 있게, 해당 파라미터가 정상 범위를 벗어났을 때 배터리 전극·전지에 어떤 영향을 미치는지 구체적으로 설명하세요. 예: 리튬 투입량 초과 시 전극 리튬 저항 증가 → 구조적 안정성 저하 → 성능 악화 등.

**권장 조치사항**:
1. 첫 번째 권장 조치 (가장 중요)
2. 두 번째 권장 조치
3. 세 번째 권장 조치

중요: LOT ID나 제목을 본문에 다시 언급하지 마세요. 권장 조치사항에는 반드시 번호를 사용하세요.

--- LOT 데이터 ---
{LOT 데이터 텍스트}

--- 분석 결과 ---
{분석 요약 텍스트}
```

## 2. 텍스트 레포트 예시 (OpenAI 응답)

```
**불량 원인 분석**: "리튬 투입량 (50.0%), 공정 시간 (30.0%), 첨가제 비율 (20.0%)"

**불량 발생 메커니즘**:
- 리튬 투입량: 이 LOT에서 리튬 투입량이 최적 범위를 초과하여 전극의 리튬 저항이 증가하고, 이는 전극 구조적 안정성을 저하시켜 배터리의 전체 성능을 악화시키는 원인이 되었습니다.
- 공정 시간: 공정 시간이 과도하게 길어져 전극의 전기화학적 특성이 저하되고, 이는 불량 배터리의 전기적 성능 저하로 이어졌습니다.
- 첨가제 비율: 첨가제 비율이 최적 범위를 벗어나면 전극의 전도성과 안정성이 저하되어, 이로 인해 배터리의 수명과 안전성이 감소하는 결과를 초래합니다.

**권장 조치사항**:
1. 리튬 투입량을 최적 범위로 조정하고, 투입 과정에서의 실시간 모니터링을 강화하여 불량 재발을 방지합니다.
2. 공정 시간에 대한 재검토를 실시하고, 표준화된 공정 조건을 수립하여 불량 발생 가능성을 최소화합니다.
3. 첨가제 비율을 최적화하고, 해당 비율에 대한 정기적인 점검 및 조정을 수행하여 품질을 유지합니다.
```

### 3-2. tables 구조 (통계 정보: 비전공자 친화적 설명)

| 상태 | 예시 문구 |
|------|----------|
| 정상(범위 내) | 정상 범위 안에 있으며, 과거 합격 LOT 평균(약 44.50 분)과 유사한 수준입니다. |
| 경고(상한 초과) | 정상 범위 상한(약 50.00 분)을 5% 초과했습니다. 합격 LOT 평균(약 44.50 분)보다 높은 편입니다. |

## 4. OpenAI API 실패 시 기본 레포트 예시

API 호출 실패 시 생성되는 폴백 텍스트:

```
**불량 원인 분석**: "리튬 투입량 (50.0%), 공정 시간 (30.0%), 첨가제 비율 (20.0%)"

**불량 발생 메커니즘**:
공정 파라미터 분석 결과, 리튬 투입량: 2.7500 kg, 첨가제 비율: 0.0823, 공정 시간: 45.23 분, 습도: 4.71%, 탱크 압력: 125.30 kPa, 소결 온도: 785.50 °C 등의 값이 측정되었습니다. 상세 메커니즘은 아래 시각화 데이터를 참고하세요.

**권장 조치사항**:
1. 정상 범위를 벗어난 파라미터를 최적 범위로 조정합니다.
2. 투입·공정 과정의 실시간 모니터링을 강화하여 불량 재발을 방지합니다.
3. 재검사 후 유효성을 확인합니다.
```

## 5. DB 저장 구조 (lot_defect_reports)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | INT | 시계열 순서 (가장 오래된 = 1) |
| lot_id | VARCHAR(100) | LOT 식별자 (PK) |
| report_content | TEXT | 위 텍스트 레포트 전체 |
| lot_data_json | JSON | lotData + visualization |
| timestamp | TIMESTAMP | 생성/갱신 시각 |
